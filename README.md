# WCFC: A Robust BrainPrint Authentication System with Neural LDPC Decoding
This framework integrates a Wavelet Scattering Transform (WST) based feature extractor (WST-CAA) with a Neural Min-Sum (Neural-MS) LDPC decoder to construct a secure and robust Fuzzy Commitment Scheme for EEG biometrics.  

## System Overview
The system operates in two main phases: **Feature Extraction** and **Key Binding/Recovery**.  
**EEG Feature Pipeline**: Raw EEG signals are preprocessed, transformed via Wavelet Scattering, and fed into a DenseNet model to extract stable binary bio-codes.  
**Neural LDPC Module**: A Neural Min-Sum decoder is trained to correct errors in the bio-codes, allowing for the reliable recovery of cryptographic keys bound to the user's brainwaves.

## Datasets
**DEAP**: The current implementation is optimized for the DEAP dataset. The DEAP dataset is a widely used resource for affective computing and EEG-based emotion recognition research. It contains electroencephalogram (EEG) and peripheral physiological signals recorded from 32 participants as they watched 40 one-minute music video excerpts. To access the raw dataset, you must submit a usage application by the official website: [DEAP](https://eecs.qmul.ac.uk/mmv/datasets/deap/download.html).

## How To Use
### Clone
Clone this project to your local machine:
```
git clone https://github.com/Lin-X-H/WCFC.git
cd WCFC
```
> [!NOTE]
> All the following commands are assumed to be executed from the project root directory.

### Data Preparation(EEG)
Before training the feature extractor, the raw DEAP dataset must be processed into tensor format.  
1. Download the **DEAP dataset** and place it in `Dataset/DEAPDataset/`.  
2. Run the **preprocessing script** to artifact removal and segmentation. This generates `.pkl` files in `EEG_Extract/Preprocessing/ProcessedData/`.  
3. Transform the processed data using **WST**. This generates `.pt` files in `EEG_Extract/WaveletScattering/AfterWST/`.  

### Feature Extractor Training
Train the DenseNet model to learn discriminative EEG features.  
```
cd EEG_Extract/train
python train.py
```
- Input: Reads `.pt` files from `AfterWST`.
- Output: Saves the best performing model to `WCFC/model/`, named with its accuracy (e.g., `Noabl(CAADN)ACC0.9922.pth`).

### LDPC Construction & Decoder Training
This module operates independently of the EEG data. It trains a neural network to decode LDPC codes under noisy conditions.  
1. **Matrix Generation**  
```
cd Neural_LDPC
python GenerateMatrix.py
```
- Function: Reads the proto-graph (Base Graph) and expands it by factor Z (Lift Size).
- Output: Generates the Generator Matrix (GM) for encoding and Parity Check Matrix (PCM) for decoding in `BaseGraph_GM/`.

2. **Train Neural Decoder**
```
python Neural_MS.py
```
- Process: Generates synthetic codewords, adds noise, and trains the Min-Sum network to recover the original message.  
- Output: Saves the trained decoder to `WCFC/model/ldpc_neural_decoder.pth`.

### Run Fuzzy Commitment
Once both models are trained, use `FuzzyCommit.py` to simulate the full authentication lifecycle.  
> [!NOTE]
> Configuration Required: Open `FuzzyCommit.py` and update `densenet_path` to match the specific filename generated by EEG model.

Run the system:
```
cd WCFC
python FuzzyCommit.py
```

**What happens**  
1. Enrollment: Extracts features from a user (User A), generates a random key, encodes it using LDPC, and XORs it with the feature to create a Commitment.  
2. Legitimate Verification: Uses a different sample from User A. The system attempts to unlock the Commitment using the Neural LDPC decoder.
3. Imposter Attack: Uses samples from Users B, C, D... to attempt to unlock User A's commitment (should fail).

**Utilities**  
To analyze the quality of the generated binary codes (Entropy, Hamming Distance distribution), you can use the diagnostic tool:
```
python feature_diagnostics.py
```
